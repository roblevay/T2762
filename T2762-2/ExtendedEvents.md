# SQL Server Extended Events – Developer Exercises

## 📘 Overview

In these exercises, you will use **Extended Events** to troubleshoot and analyze behavior in your own database code.

Focus:

* Identify slow queries
* Catch runtime errors
* Understand concurrency issues (deadlocks)

All sessions are created manually in SSMS (no wizard).

---

# 🧪 Exercise 1: Find Problematic Queries

## 🎯 Goal

Identify **queries in your application that perform poorly**.

Instead of just “long queries”, think:

> *Which statements in my code need optimization?*

---

## 🛠️ Steps

1. Create a session: `SlowQueries_Dev`

2. Add event:

   * `sql_statement_completed`

3. Add filter:

   * `duration > 2000000` (2 seconds)

4. Add Actions (very important for developers):

   * `sql_text`
   * `database_name`
   * `client_app_name`
   * `username`

5. Add `event_file` target
   Example:

   ```
   C:\XELogs\SlowQueries_Dev.xel
   ```

6. Start the session

---

## 🧪 Test Scenario

Start  **Watch Live Data**

Run a deliberately inefficient query:

```sql
USE AdventureWorks

SELECT *
FROM Production.Product AS o1
CROSS JOIN  Production.Product AS o2

```

Another:
```sql
WAITFOR DELAY '00:00:03'
```
---

## ✅ What to look for

In **Watch Live Data**:

* Which query was captured?
* How long did it run?
* What application/user executed it?

---

## 💡 Developer Insight

Ask yourself:

* Could this query use indexes?
* Is it returning too much data?
* Can it be rewritten?

# 🧪 Exercise 2: Catch Runtime Errors in Code

## 🎯 Goal

Capture **errors generated by your T-SQL code**

👉 This is much more useful for developers than failed logins.

---

## 🛠️ Steps

1. Create session: `QueryErrors_Dev`

2. Add event:

   * `error_reported`

3. Add filter:

   * `severity >= 11` (ignore informational messages)

4. Add Actions:

   * `sql_text`
   * `database_name`
   * `username`

5. Add `event_file` target:

   ```
   C:\XELogs\QueryErrors_Dev.xel
   ```

6. Start session

---

## 🧪 Test Scenario

Run queries that generate errors:

```sql
-- Divide by zero
SELECT 1 / 0;

-- Invalid object
SELECT * FROM NonExistingTable;
```

---

## ✅ What to look for

* Which error was captured?
* What SQL caused it?
* Can you correlate error ↔ code?

---

## 💡 Developer Insight

This is useful for:

* debugging stored procedures
* catching hidden runtime errors
* logging production issues

# 🧪 Exercise 3: Understand Deadlocks (Concurrency Bugs)

## 🎯 Goal

Capture and analyze **deadlocks caused by application logic**

👉 This is a *developer problem*, not just admin.

---

## 🛠️ Steps

1. Create session: `Deadlocks_Dev`

2. Add event:

   * `xml_deadlock_report`

3. Add Actions:

   * `sql_text`
   * `database_name`

4. Add `event_file` target:

   ```
   C:\XELogs\Deadlocks_Dev.xel
   ```

5. Start session

---

## 🧪 Test Scenario: Simulate a Deadlock

### Setup

```sql
USE tempdb;

DROP TABLE IF EXISTS DeadlockTest;

CREATE TABLE DeadlockTest (
    ID INT PRIMARY KEY,
    Value VARCHAR(100)
);

INSERT INTO DeadlockTest (ID, Value)
VALUES (1, 'First'), (2, 'Second');
```

---

### Session A

```sql
BEGIN TRAN;

UPDATE DeadlockTest SET Value = 'A1' WHERE ID = 1;

WAITFOR DELAY '00:00:10';

UPDATE DeadlockTest SET Value = 'A2' WHERE ID = 2;

COMMIT;
```

---

### Session B

```sql
BEGIN TRAN;

UPDATE DeadlockTest SET Value = 'B1' WHERE ID = 2;

WAITFOR DELAY '00:00:10';

UPDATE DeadlockTest SET Value = 'B2' WHERE ID = 1;

COMMIT;
```

---

## ✅ What to look for

* Which query was the deadlock victim?
* Which resources were locked?
* What order caused the problem?

---

## 💡 Developer Insight

Ask:

* Can I access rows in a consistent order?
* Can I reduce transaction scope?
* Can I use retry logic?
